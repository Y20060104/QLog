================================================================================
                          核心规范概览
================================================================================

【编码规范】(RULES.md)
  • C++20 标准，禁用异常和 RTTI
  • 命名约定: snake_case, 成员变量尾下划线
  • 常量: k_snake_case 或 UPPER_CASE
  • 行限: 100 字符（clang-format 自动化）

【核心设计原则】(RULES.md)
  • Zero-Copy: Hot Path 不分配堆内存
  • Lock-Free: Hot Path 禁止 mutex（仅用 acquire/release）
  • Deferred Formatting: 热路径只做二进制序列化
  • Cache-Line Alignment: 64 bytes 对齐防 false sharing
  • Hot Path < 300ns: 单次日志调用时间约束

【Milestone 规划】(STATE.md)
  M0 无锁原语          (Week 2)
  M1 SPSC Ring Buffer  (Week 2-3)
  M2 MPSC Ring Buffer  (Week 3)
  M3 双路调度器        (Week 3-4)
  M4 序列化            (Week 4)
  M5 格式化引擎        (Week 5)
  M6 Appender 体系     (Week 5-6)
  M7 Worker 线程       (Week 6-7)
  M8 管理器            (Week 7)
  M9-M12 可选模块      (Week 8-12)

【性能目标】(STATE.md)
  • 单次日志调用: < 300ns (Release)
  • 单线程吞吐量: > 5M entries/s
  • 10线程吞吐量: > 20M entries/s
  • 内存占用: < 2MB (10 线程 × 200 万条)
  • 压缩率: < 15% vs 文本

================================================================================
                      重要注意事项
================================================================================

1. 【编码约束】
   • 热路径（Hot Path 定义在 RULES.md）不允许：
     - malloc/new/delete 任何分配
     - mutex/lock（仅 atomic + acquire/release）
     - 虚函数指针间接调用

2. 【内存安全】
   • 所有并发代码必须通过 ThreadSanitizer
   • 使用脚本: ./scripts/run_sanitizers.sh thread
   • TSan 发现竞争即为 blockers，必须解决

3. 【性能要求】
   • Hot Path 基准必须 < 300ns (Release)
   • 使用: ./benchmark/cpp/bench_*.cpp
   • 对标目标: BqLog 5M entries/s

4. 【版本控制】
   • 每个 Milestone 完成后创建 tagged commit
   • 提交前运行 format_code.sh
   • 编译命令需在 CMakeLists.txt 中记录
